Cross-Namespace Arrows

  • CSI Driver → mcp-demo pods: Mounts Workload API socket into MCP Client, MCP Server, Test Runner
  • OIDC Discovery → Keycloak: Keycloak's SPIFFE SPI fetches JWKS from OIDC Discovery Provider's HTTP proxy
  • MCP Client → Keycloak: Posts token requests with JWT SVID as client_assertion
  • Controller Manager → mcp-demo: Watches ClusterSPIFFEID CRDs, auto-registers entries for pods

Key point: The signing key used by SPIRE Server is never shared with Keycloak. Keycloak only sees the public key via JWKS. This is the same trust model as any OIDC provider.

Docker Build Pipeline

The custom Keycloak image is built in a 4-stage multi-stage Dockerfile:

# Stage 1: Build SVID Client Authenticator SPI (Java/Maven)
FROM maven:3.9-eclipse-temurin-17 AS build-svid-auth
# Clone + mvn package

# Stage 2: Build DCR SPI (Java/Maven)
FROM maven:3.9-eclipse-temurin-17 AS build-dcr
# Clone + mvn package

# Stage 3: Build SPIRE Software Statements Plugin (Go 1.22)
FROM golang:1.22 AS build-spire-plugin
# Clone + make build

# Stage 4: Final Keycloak image with all SPIs
FROM quay.io/keycloak/keycloak:26.2.5
COPY --from=build-svid-auth .../spiffe-svid-client-authenticator-*.jar /opt/keycloak/providers/
COPY --from=build-dcr .../spiffe-dcr-spi-*.jar /opt/keycloak/providers/
RUN /opt/keycloak/bin/kc.sh build
