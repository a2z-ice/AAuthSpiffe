Step-by-Step Deployment Guide

Five ordered phases, each with exact commands, annotated configs, and validation gates.

Phase Dependencies

Deployment Phase Flow
Phase 1: Infrastructure
Kind cluster with 2 nodes + NodePort mappings
Phase 2: SPIRE Stack
SPIRE Server, Agent, OIDC, CSI, Controller Manager
Phase 3: Custom Builds
keycloak-spiffe:latest loaded into Kind
Phase 4: Keycloak + Data
Keycloak serving at :30080, realm imported
Phase 5: Workloads + Validation
MCP pods running, test-spiffe-auth.sh passes

Phase 1: Infrastructure

Create the Kind cluster with NodePort mappings for host access.

1.1 Kind Cluster Config

# infra/kind-config.yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
name: spiffe-mcp-demo
nodes:
  - role: control-plane
    extraPortMappings:
      - containerPort: 30080    # Keycloak
        hostPort: 30080
        protocol: TCP
      - containerPort: 30443    # OIDC Discovery
        hostPort: 30443
        protocol: TCP
  - role: worker
    labels:
      spire-agent: "true"

1.2 Create Cluster

# Idempotent: check before creating
kind get clusters | grep -q spiffe-mcp-demo || kind create cluster --config infra/kind-config.yaml
kubectl cluster-info --context kind-spiffe-mcp-demo

1.3 Validation Gate

# Must show 2 nodes in Ready state
kubectl get nodes --context kind-spiffe-mcp-demo
# Expected: control-plane Ready, worker Ready
Troubleshooting: If kind create cluster fails with Docker errors, ensure Docker Desktop is running and has enough resources allocated.

Phase 2: SPIRE Stack

Deploy SPIRE via Helm hardened charts.

2.1 Add Helm Repos

helm repo add spiffe https://spiffe.github.io/helm-charts-hardened/
helm repo update

2.2 SPIRE Values

# infra/spire-values.yaml (key sections)
global:
  spire:
    clusterName: "spiffe-mcp-demo"
    trustDomain: "example.org"
    jwtIssuer: "https://oidc-discovery.example.org"

spire-server:
  enabled: true
  replicaCount: 1
  persistence:
    type: pvc
    size: 1Gi
  controllerManager:
    enabled: true

spire-agent:
  enabled: true

spiffe-csi-driver:
  enabled: true

spiffe-oidc-discovery-provider:
  enabled: true
  config:
    domains:
      - "oidc-discovery.example.org"
      - "spiffe-oidc-discovery-provider.spire-system.svc.cluster.local"

2.3 Install SPIRE

# Install CRDs first
helm upgrade --install --create-namespace -n spire-system \
  spire-crds spire-crds \
  --repo https://spiffe.github.io/helm-charts-hardened/ --wait

# Install SPIRE stack
helm upgrade --install -n spire-system \
  spire spire \
  --repo https://spiffe.github.io/helm-charts-hardened/ \
  --values infra/spire-values.yaml \
  --wait --timeout 300s

2.4 Validation Gate

# All pods Running
kubectl -n spire-system get pods

# Server health
kubectl -n spire-system exec deploy/spire-server -- \
  /opt/spire/bin/spire-server healthcheck

# OIDC Discovery responds (from host)
curl -sk https://localhost:30443/.well-known/openid-configuration | jq .issuer
Troubleshooting: SPIRE Agent CrashLoopBackOff usually means the Server isn't ready yet. Wait 60s and check server health first. Helm install timeout often means PVC provisioning issues — check kubectl get sc.

Phase 3: Custom Builds

Build the custom Keycloak image with SPIFFE SPIs and load into Kind.

3.1 Build Custom Images

# Build Keycloak with SPIFFE SPIs (multi-stage, ~5-10 min first time)
docker build -f docker/Dockerfile.keycloak-custom -t keycloak-spiffe:latest .

# Build SPIRE plugin image
docker build -f docker/Dockerfile.spire-plugin -t spire-software-statements:latest .

3.2 Load into Kind

# Load images (no registry needed)
kind load docker-image keycloak-spiffe:latest --name spiffe-mcp-demo
kind load docker-image spire-software-statements:latest --name spiffe-mcp-demo

3.3 Validation Gate

docker images | grep keycloak-spiffe
docker images | grep spire-software-statements

Phase 4: Keycloak + Data

Deploy PostgreSQL, Keycloak, and configure the OIDC NodePort.

4.1 Deploy PostgreSQL

kubectl apply -f manifests/postgres.yaml
kubectl -n mcp-demo wait --for=condition=ready pod -l app=postgres --timeout=120s

4.2 Deploy Keycloak

# Keycloak imports realm-config.json on first boot
kubectl apply -f manifests/keycloak.yaml
kubectl -n mcp-demo wait --for=condition=ready pod -l app=keycloak --timeout=300s

# OIDC Discovery NodePort for host access
kubectl apply -f manifests/oidc-nodeport.yaml

4.3 Configure Keycloak SPIFFE Flow

# Run the configuration script (creates spiffe-clients auth flow)
./scripts/configure-keycloak.sh

This script:

  1. Gets an admin token from Keycloak
  2. Creates a "spiffe-clients" authentication flow (copy of "clients")
  3. Adds the client-spiffe-jwt authenticator execution to the flow
  4. Binds the realm to use the spiffe-clients flow
  5. Updates JWKS URLs on the realm and SPIFFE client

4.4 Validation Gate

# Keycloak health
curl -s http://localhost:30080/realms/master | jq .realm

# Realm OIDC
curl -s http://localhost:30080/realms/mcp-demo/.well-known/openid-configuration | jq .issuer

# Admin console accessible at http://localhost:30080/admin (admin/admin)
Troubleshooting: ImagePullBackOff on keycloak-spiffe means the image wasn't loaded into Kind. Re-run kind load docker-image keycloak-spiffe:latest --name spiffe-mcp-demo. If Keycloak shows "Connection refused" to postgres, wait for PostgreSQL readiness.

Phase 5: Workloads + Validation

Deploy MCP workloads, create SPIFFE IDs, and run the full test suite.

5.1 Create ClusterSPIFFEIDs

kubectl apply -f manifests/cluster-spiffe-ids.yaml
# ClusterSPIFFEID for mcp-client
apiVersion: spire.spiffe.io/v1alpha1
kind: ClusterSPIFFEID
metadata:
  name: mcp-client-id
spec:
  className: "default"
  spiffeIDTemplate: "spiffe://example.org/ns/{{ .PodMeta.Namespace }}/sa/{{ .PodSpec.ServiceAccountName }}"
  podSelector:
    matchLabels:
      app: mcp-client
  jwtTTL: "5m"
  namespaceSelector:
    matchLabels:
      kubernetes.io/metadata.name: mcp-demo

5.2 Deploy MCP Workloads

kubectl apply -f manifests/mcp-client.yaml
kubectl apply -f manifests/mcp-server.yaml
kubectl -n mcp-demo wait --for=condition=ready pod -l app=mcp-client --timeout=120s

5.3 Run the Demo

# Run the SPIFFE authentication demo script
kubectl -n mcp-demo exec -it deploy/mcp-client -- bash /scripts/test-spiffe-auth.sh

5.4 Run Full Test Suite

# Deploy test runner and run all 80 tests
kubectl apply -f manifests/test-runner.yaml
./scripts/run-all-tests.sh

5.5 Validation Gate

# MCP client can fetch SVID
kubectl -n mcp-demo exec deploy/mcp-client -- \
  python3 -c "
from spiffe import WorkloadApiClient
c = WorkloadApiClient('unix:///spiffe-workload-api/spire-agent.sock')
s = c.fetch_jwt_svid(audience={'test'})
print(s.spiffe_id)
"
# Should print: spiffe://example.org/ns/mcp-demo/sa/mcp-client

One-Command Alternative

If you prefer to deploy everything at once:

# Runs all 5 phases sequentially with validation
./deploy-all.sh

# Teardown when done
./teardown.sh
