What This Project Does

This system demonstrates how AI agents (MCP clients) can authenticate to an OAuth Authorization Server (Keycloak) using cryptographically verifiable SPIFFE identities instead of long-lived client secrets. It follows the IETF draft draft-ietf-oauth-spiffe-client-auth and runs entirely inside a Kind (Kubernetes-in-Docker) cluster on macOS.

Key Innovation Instead of storing and rotating client_secret values, workloads prove their identity with short-lived JWT SVIDs (5-minute TTL) that are automatically issued, cryptographically signed, and verified through a distributed trust chain.

Quick Start

Prerequisites: Docker Desktop, kind, kubectl, helm, jq installed on macOS. 
# Verify prerequisites
docker version && kind version && kubectl version --client && helm version && jq --version

# Deploy everything (takes ~10 minutes)
./deploy-all.sh

# Run the SPIFFE authentication demo
kubectl -n mcp-demo exec -it deploy/mcp-client -- bash /scripts/test-spiffe-auth.sh

# Run the full test suite (80 tests)
./scripts/run-all-tests.sh

# Teardown
./teardown.sh
